# 2-Minute Live Demo Script

## 0:00 - 0:20 (Frame)

"This is **PRs for Operations**. What pull requests do for code, this does for operations in spreadsheets. It is not spreadsheet chat."

## 0:20 - 0:45 (Messy Input + Prompt)

"Here’s a messy receivables tab with non-standard headers. I’ll give one sentence: *We're behind on collections. Build today's collections queue and fix the follow-ups.*"

Click **Propose Changes**.

## 0:45 - 1:20 (Review Layer)

"The agent maps messy headers to business concepts, then proposes a structured patch set. Every change is reviewable by row/field before writeback."

Show patch rows with `before -> after` and reasons.
Optionally deselect one patch to show human control.

## 1:20 - 1:45 (Apply)

Click **Apply Approved**.

"Now we apply only approved patches to the raw sheet, then generate execution artifacts."

## 1:45 - 2:00 (Operational Output)

Open `Collections Queue` and `Report` tabs.

"Queue is executable work for collectors. Report is audit-ready: what changed, why, and projected recoverable cash."

Close with: "PRs for Operations turns messy spreadsheet context into reviewable operational patches."
